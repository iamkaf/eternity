The Problem

RPG Maker's Limits

Proprietary. No version control. Single-file database locks. No multiplayer. No plugin API. Tied to NW.js. Limited export targets.

Eternity's Answer

Open-source. One entity per file (Git-friendly). WebGPU renderer. Relay multiplayer. Full plugin API. Export anywhere.

Three Core Patterns

Schema

Zod Schema Registry

Single source of truth. Validates data, generates editor forms, powers localization extraction. Plugins register schemas — the editor auto-generates UI.

Command

Command Pattern

Every mutation is a Command with execute() and undo(). Full undo/redo. Disk sync. Multiplayer state replication. One pattern, everywhere.

Catalog

json-render Catalog

All editor panels — built-in and plugin — are catalog entries constrained by Zod schemas. Plugins extend UI without shipping raw React components.

Multiplayer

Architecture

Authoritative WebSocket relay server. Same engine runs headlessly via HeadlessPlatform. MessagePack binary protocol. Client-side prediction for responsiveness.

Distribution

Server exports as a Node.js Single Executable Application — one binary, no runtime dependencies. Download, run, host. Game data embedded via sea.getAsset().

Export Pipeline

V1

Electron Desktop

Windows, macOS, Linux. ASAR packaging. Auto-update via blockmap diffing. Code signing.

V1

Web / itch.io

Static site export. PWA with Workbox service worker. IndexedDB saves. Offline play.

Future

Tauri / Mobile

3-10MB bundles. Native Android/iOS via Tauri v2. PAL makes it a clean addition.

Asset optimization: PNG → WebP (25-35% savings), Audio → Opus, unused file exclusion, optional AES-256 encryption.
